🗂️ קבצים בתיקייה זו

שלושת המקורות האלה שייכים לערכת ניוד AS/400 → UNIX. כל תוכנית נושאת שני מסלולי קוד — שורות המסומנות ב-X&FLTA מהודרות עבור AS/400, שורות המסומנות ב- X&FLTU מהודרות עבור UNIX. אלה כלי עזר עמיתים של אותה ערכה, ולא שרשרת קריאה ישירה ביניהם.

קובץסוגשפהמטרה
UNRDRADD.cbl תת-תוכנית שירות (נקראת) COBOL (ILE, AS/400 ו-UNIX) רוטינת READ גנרית לקובץ מפתחי עבור הקובץ המאונדקס UNRECADD. נקראת עם קוד פעולה + מפתח, מחזירה רשומה אחת + סטטוס. בונה את שם הקובץ הפיזי באופן דינמי.
UNCHKWEB#P.rpg תת-תוכנית שירות (נקראת) RPG III / RPG 400 (פורמט קבוע) בדיקת סביבה והרשאות. מזהה מצב AS/400 מול WEB, טוענת את טבלת ההרשאות מאזורי נתונים, מתחזקת דגל "כניסה-30", מחזירה מצב + קוד הרשאה.
UNCVTD2C.cl תוכנית משגר / מנהל CL (Control Language) משגר "המרת קובץ ל-CSV". קורא לתוכנית החיצונית UNFIL2CSV ושולח בחזרה את כתובת ה-IFS של המסמך שנוצר כהודעת תוכנית.

יחסים — מי מריץ את מי

הערה כנה: שלוש התוכניות הן כלי עזר עצמאיים של אותה ערכה. אין שרשרת קריאה ישירה ביניהן. UNCVTD2C קורא לתוכנית החיצונית UNFIL2CSV (אינה קיימת בתיקייה זו). UNRDRADD היא רוטינת שירות שתוכניות ברמה גבוהה יותר קוראות לה. UNCHKWEB#P נקראת על ידי יישומים/תפריטים הזקוקים למצב הסביבה + מצב ההרשאות.

📘 UNRDRADD.cbl — רוטינת READ לקובץ מפתחי

שם התוכנית ומשמעותו

UNRDRADD = UN (קידומת הערכה) + RD (read) + RADD (קובץ כתובות הרשומות UNRECADD) — רוטינת READ גנרית לקובץ UNRECADD.

כותרת עברית: רוטינת READ קובץ UNRECADD — בונה שם דינמי לקובץ — "רוטינת READ לקובץ UNRECADD — בונה את שם הקובץ באופן דינמי."

תפקיד עסקי: עוטף שיטת גישה לשימוש חוזר סביב הקובץ המאונדקס UNRECADD. תוכניות ברמה גבוהה מעבירות קוד פעולה (UN-PEULA) בתוספת מפתח תלת-חלקי (ספרייה + קובץ + שם רשומה) ומקבלות בחזרה רשומה אחת (LINK-REC) בתוספת סטטוס (UN-RETURN). הקובץ נפתח פעם אחת ושמו הפיזי נבנה באופן דינמי (<file>.RAD, לעיתים עם קידומת נתיב IFS ממשתנה הסביבה TISTEMP_DDS ב-UNIX).

PROCEDURE DIVISION — כל פסקה

CBL 000-MONITOR — מנתב ראשי / נקודת כניסה
פרמטרים: USING UN-PEULA, UN-RETURN, LINK-KEY, LINK-REC
נקרא על ידי: כניסה לתוכנית
קורא ל: ZA00-INIT002-CLOSE001-INIT100-MAIN

תיאור: מנהל הרוטינה. בתצוגה הראשונה ב-UNIX הוא מבצע ZA00-INIT. אם הפעולה היא סגירה (UN-PEULA-CLOSE-FILE), הוא סוגר את הקובץ הפתוח ויוצא. אחרת הוא מבצע 001-INIT (פתיחה חד-פעמית + שמירת מפתח) ואז 100-MAIN (ניתוב), ואז EXIT PROGRAM.

IF UN-PEULA-CLOSE-FILE
   IF SW-FILE-OPEN-ON  PERFORM 002-CLOSE  END-IF
   GO TO 998
END-IF.
PERFORM 001-INIT.
PERFORM 100-MAIN.
998. EXIT PROGRAM.
CBL 001-INIT — פתיחה חד-פעמית ושמירת מפתח
נקרא על ידי: 000-MONITOR
קורא ל: 002-CLOSE (UNIX, אם הספרייה/קובץ השתנו) 003-BUILD-NAME-DATABASE (UNIX)

תיאור: פותח את הקובץ בפעם הראשונה (בונה את שמו הדינמי ב-UNIX), מאפס את UN-RETURN ל-'00', ומעתיק את חלקי ה-LINK-KEY של הקורא אל מפתח הרשומה. ב-UNIX הוא פותח מחדש אם הספרייה/קובץ השתנו מאז הקריאה הקודמת.

IF SW-FILE-OPEN-OFF
   *> PERFORM 003-BUILD-NAME-DATABASE (UNIX)
   OPEN INPUT UNRECADD-FILE
   SET SW-FILE-OPEN-ON TO TRUE
END-IF.
MOVE '00' TO UN-RETURN.
MOVE LINK-KEY-LIB-NAME  TO UN-RADD-LIB-NAME.  *> +file +rec
CBL 002-CLOSE — סגירת הקובץ
נקרא על ידי: 000-MONITOR001-INIT (UNIX)
קורא ל: —

תיאור: מגדיר SW-FILE-OPEN-OFF ומבצע CLOSE UNRECADD-FILE.

SET SW-FILE-OPEN-OFF TO TRUE.
CLOSE UNRECADD-FILE.
CBL 003-BUILD-NAME-DATABASE — שם קובץ דינמי (UNIX)
נקרא על ידי: 001-INIT (מסלול UNIX)
קורא ל: —

תיאור: בונה את שם הקובץ הפיזי כ-<file>.RAD אל EZ-DATABASE. הוא קורא את משתנה הסביבה TISTEMP_DDS אל LIB-PATH; אם ריק הוא משתמש בשם הפשוט, אחרת הוא מוסיף את נתיב ה-IFS כקידומת (path/<file>.RAD) אל DATABASE-UNRECADD.

STRING LINK-KEY-FILE-NAME DELIMITED BY SPACE
       '.RAD' DELIMITED BY SIZE INTO EZ-DATABASE.
*> DISPLAY "TISTEMP_DDS" UPON ENVIRONMENT-NAME (UNIX)
IF LIB-PATH = SPACE   MOVE EZ-DATABASE TO DATABASE-UNRECADD
ELSE STRING LIB-PATH '/' EZ-DATABASE INTO DATABASE-UNRECADD END-IF.
CBL 100-MAIN — מנתב פעולות
נקרא על ידי: 000-MONITOR
קורא ל: 104-READ-RANDOM102-START-EQUAL101-START-NOT-LESS103-START-GRATER107-READ-LAST111-READ-NEXT-MOVE112-READ-PRIOR-MOVE200-MOVE

תיאור: מסתעף לפעולה המבוקשת דרך GO TO … DEPENDING ON UN-PEULA. קוד מחוץ לטווח נופל אל 000-TAKALA שמגדיר UN-RETURN-ERR-PEULA. עבור צירופי START+READ הוא קורא רק אם ה-START החזיר תקין (UN-RETURN-TAKIN).

GO TO 001-READ-RANDOM 002-START-EQUAL-READ
      003-START-NOT-LESS-READ 004-READ-NEXT 005-READ-PRIOR
      006-READ-LAST 007-START-NOT-LESS 008-START-GRATER
   DEPENDING ON UN-PEULA.
000-TAKALA. SET UN-RETURN-ERR-PEULA TO TRUE. GO TO 999.
CBL 101-START-NOT-LESS — מיקום על מפתח ≥
נקרא על ידי: 100-MAIN (פעולות 3, 7)
קורא ל: —

תיאור: START ממקם את הסמן על הרשומה הראשונה שמפתחה אינו קטן מ- (≥) המפתח המבוקש. ב-INVALID KEY מגדיר החזרה '01'.

START UNRECADD-FILE KEY IS NOT < UN-RADD-REC-KEY
      INVALID KEY MOVE '01' TO UN-RETURN
END-START.
CBL 102-START-EQUAL — מיקום על מפתח =
נקרא על ידי: 100-MAIN (פעולה 2)
קורא ל: —

תיאור: START ממקם את הסמן על הרשומה שמפתחה שווה ל-מפתח המבוקש. INVALID KEY ← החזרה '01'.

START UNRECADD-FILE KEY IS EQUAL UN-RADD-REC-KEY
      INVALID KEY MOVE '01' TO UN-RETURN
END-START.
CBL 103-START-GRATER — מיקום על מפתח >
נקרא על ידי: 100-MAIN (פעולה 8)
קורא ל: —

תיאור: START ממקם את הסמן על הרשומה הראשונה שמפתחה גדול ממש מהמפתח המבוקש. INVALID KEY ← החזרה '01'.

START UNRECADD-FILE KEY IS > UN-RADD-REC-KEY
      INVALID KEY MOVE '01' TO UN-RETURN
END-START.
CBL 104-READ-RANDOM — קריאה אקראית לפי מפתח
נקרא על ידי: 100-MAIN (פעולה 1)
קורא ל: —

תיאור: קורא את הרשומה שמפתחה שווה למפתח המבוקש. INVALID KEY (לא נמצא) ← החזרה '01'.

READ UNRECADD-FILE
     INVALID KEY MOVE '01' TO UN-RETURN
END-READ.
CBL 105-READ-NEXT — קריאת הבא (+ טיפול ב-EOF)
נקרא על ידי: 111-READ-NEXT-MOVE107-READ-LAST-UNIX
קורא ל: 113-START-SOF (ב-EOF)

תיאור: קורא את הרשומה הרציפה הבאה. AT END ← החזרה '02' (EOF) ועיגון מחדש דרך 113-START-SOF. ב-UNIX, סטטוס קובץ '9D' ממופה להחזרה '05' (נעילה).

READ UNRECADD-FILE NEXT
     AT END MOVE '02' TO UN-RETURN
            PERFORM 113-START-SOF
END-READ.
*> IF FILE-STATUS = '9D' MOVE '05' (lock) — UNIX
CBL 106-READ-PRIOR — קריאת הקודם (+ טיפול ב-BOF)
נקרא על ידי: 112-READ-PRIOR-MOVE
קורא ל: 114-START-BEGIN (ב-BOF)

תיאור: קורא את הרשומה הקודמת (PRIOR ב-AS/400, PREVIOUS ב-UNIX). AT END (תחילת קובץ) ← החזרה '03' ועיגון מחדש דרך 114-START-BEGIN.

READ UNRECADD-FILE PRIOR   *> PREVIOUS on UNIX
     AT END MOVE '03' TO UN-RETURN
            PERFORM 114-START-BEGIN
END-READ.
CBL 107-READ-LAST — קריאת הרשומה האחרונה
נקרא על ידי: 100-MAIN (פעולה 6)
קורא ל: 107-READ-LAST-UNIX (UNIX)

תיאור: ב-AS/400 משתמש ב-READ … LAST RECORD; AT END ← החזרה '01'. ב-UNIX הוא מאציל ל-107-READ-LAST-UNIX (LAST אינו נתמך ישירות).

READ UNRECADD-FILE LAST RECORD          *> AS/400
     AT END MOVE '01' TO UN-RETURN
END-READ.
*> PERFORM 107-READ-LAST-UNIX (UNIX)
CBL 107-READ-LAST-UNIX — קריאת אחרון, מדומה (UNIX)
נקרא על ידי: 107-READ-LAST
קורא ל: 108-START-LESS-UNIX105-READ-NEXT

תיאור: מדמה "קריאת אחרון": מגדיר את המפתח ל-HIGH-VALUE, ממקם עם 108-START-LESS-UNIX (מפתח < ערך גבוה = הרשומה האחרונה), ואם תקין, קורא את הבא כדי לנחות עליה.

MOVE HIGH-VALUE TO UN-RADD-REC-KEY.
PERFORM 108-START-LESS-UNIX.
IF UN-RETURN-TAKIN  PERFORM 105-READ-NEXT  END-IF.
CBL 108-START-LESS-UNIX — מיקום על מפתח < (UNIX)
נקרא על ידי: 107-READ-LAST-UNIX
קורא ל: —

תיאור: UNIX בלבד. START ממקם את הסמן על הרשומה האחרונה שמפתחה קטן מ-המפתח המבוקש (כאן HIGH-VALUE). INVALID KEY ← החזרה '01'.

START UNRECADD-FILE KEY IS LESS THEN UN-RADD-REC-KEY
      INVALID KEY MOVE '01' TO UN-RETURN
END-START.
CBL 111-READ-NEXT-MOVE — קריאת הבא ואז מסירה
נקרא על ידי: 100-MAIN (פעולות 2,3,4)
קורא ל: 105-READ-NEXT200-MOVE

תיאור: צירוף נוחות: בצע 105-READ-NEXT ואז 200-MOVE (מעתיק את הרשומה אל LINK-REC כשתקין).

PERFORM 105-READ-NEXT.
PERFORM 200-MOVE.
CBL 112-READ-PRIOR-MOVE — קריאת הקודם ואז מסירה
נקרא על ידי: 100-MAIN (פעולה 5)
קורא ל: 106-READ-PRIOR200-MOVE

תיאור: צירוף נוחות: בצע 106-READ-PRIOR ואז 200-MOVE.

PERFORM 106-READ-PRIOR.
PERFORM 200-MOVE.
CBL 113-START-SOF — עיגון מחדש לאחר EOF
נקרא על ידי: 105-READ-NEXT
קורא ל: —

תיאור: לאחר EOF, קורא מחדש את LAST RECORD (AS/400) כדי להשאיר את הסמן על השורה האחרונה כך שקריאת-קודם עוקבת תעבוד.

READ UNRECADD-FILE LAST RECORD
     AT END GO TO 999
END-READ.
CBL 114-START-BEGIN — עיגון מחדש לאחר BOF
נקרא על ידי: 106-READ-PRIOR
קורא ל: —

תיאור: לאחר תחילת-קובץ, מרוקן את חלקי המפתח, ממקם על הרשומה הראשונה (מפתח ≥ רווחים) וקורא את הבא, כך שהסמן יושב בראש.

MOVE SPACES TO UN-RADD-LIB/FILE/REC-NAME.
START UNRECADD-FILE KEY IS NOT < UN-RADD-REC-KEY
      INVALID KEY GO TO 999 END-START.
READ UNRECADD-FILE NEXT AT END GO TO 999 END-READ.
CBL 200-MOVE — מסירת הרשומה לקורא
נקרא על ידי: 100-MAIN111 / 112 -*-MOVE

📙 UNCVTD2C.cl — משגר המרת קובץ ל-CSV

שם התוכנית ומשמעותו

UNCVTD2C = UN (הערכה) + CVT (convert) + D2C (data / document → CSV) — משגר CL דק ל"המרת קובץ ל-CSV."

תפקיד עסקי: מנהל CL מינימלי. בהינתן קובץ, ספרייה ושם מסמך הוא קורא לתוכנית החיצונית UNFIL2CSV, מקבל את כתובת ה-IFS של המסמך שנוצר, ושולח אותה בחזרה כהודעת תוכנית. הוא אינו מבצע את ההמרה בעצמו.

משתנים

משתנהסוגמשמעות בעבריתתיאור
&FILE*CHAR 10שם קובץ (קלט)פרמטר PGM — קובץ המקור להמרה.
&LIB*CHAR 10ספרייה (קלט)פרמטר PGM — הספרייה המכילה את הקובץ.
&DOCNAM*CHAR 30שם מסמך (קלט)פרמטר PGM — שם ה-CSV/מסמך היעד; מועבר הלאה אל UNFIL2CSV.

&RET*CHAR 1דגל החזרה (עבודה)מנוקה ואז נקבע על ידי UNFIL2CSV; נלכד אך אינו נבדק כאן.

PROCEDURE — שלבים מתויגים

CL START — הצהרות וממשק PGM
פרמטרים: PGM(&FILE &LIB &DOCNAM)
נקרא על ידי: מפעיל / תוכנית קוראת
קורא ל: —

תיאור: מצהיר על שלושת הפרמטרים המתקבלים בתוספת משתני העבודה &ADDR (120) ו-&RET (1).

PGM (&FILE &LIB &DOCNAM)
   DCL &RET *CHAR 1
   DCL &FILE *CHAR 10
   DCL &LIB  *CHAR 10
   DCL &DOCNAM *CHAR 30
   DCL &ADDR *CHAR 120
CL CALL — ניקוי משתני עבודה והרצת UNFIL2CSV
נקרא על ידי: START (נפילה-דרך)
קורא ל: UNFIL2CSV (חיצוני)

תיאור: מנקה את &ADDR ואת &RET, ואז קורא לממיר החיצוני UNFIL2CSV ומעביר את כל חמשת הפרמטרים. UNFIL2CSV מחזיר את נתיב ה-IFS של המסמך ב-&ADDR וסטטוס ב-&RET.

CALL:
   CHGVAR &ADDR ' '
   CHGVAR &RET  ' '
   CALL PGM(UNFIL2CSV) PARM(&FILE &LIB &DOCNAM &ADDR &RET)
CL SNDPGMMSG — דיווח כתובת המסמך; EXIT
נקרא על ידי: CALL (נפילה-דרך)
קורא ל: —

תיאור: שולח את כתובת ה-IFS של המסמך בחזרה לקורא כהודעת תוכנית, ואז ENDPGM בתווית EXIT.

   SNDPGMMSG MSG('Document''s address : ' || &ADDR)
EXIT:
   ENDPGM
🔎 &RET נלכד אך אינו נבדק במשגר זה — הערך שהתוכנית חושפת לקורא הוא &ADDR (נתיב ה-IFS של המסמך).
