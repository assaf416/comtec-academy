שלוש התוכניות הכבדות: מנהל תפריט ב-RPG, כלי בניית DDS ב-CL, ומנוע מסכי WEB ב-COBOL עם 170 סקשנים. קוד המקור נושא קוד ASCII עם הערות בעברית ב-Windows-1255 (פוענחו כאן ל-UTF-8).

🗂️ קבצים בתיקייה זו

שלוש תוכניות שאינן קשורות בזמן ריצה אך קשורות רעיונית, מתוך תת-מערכת הניוד AS/400↔WEB. כל תוכנית תומכת במסלול קוד כפול: שורות המסומנות X&FLTA מהודרות ל-AS/400 (COBOL), ושורות המסומנות X&FLTU מהודרות ל-UNIX.

קובץסוגשפהמטרה
M.rpg תוכנית אינטראקטיבית (WORKSTN) RPG III / RPG 400 מנהל התפריט הראשי של אפליקציית "ME". מציג תפריט 5250, קורא את בחירת המשתמש ומקשי הפונקציה, בודק הרשאות, מתעד פעילות, ומפעיל את התוכנית או תת-התפריט הנבחרים. 68 שגרות משנה.
UNFILE2PCC.cl תוכנית שירות CL (Control Language) כלי לבניית DDS / קבצים. מבצע הנדסה לאחור לקובץ פיזי (DSPFD/DSPFFD), יוצר חברי מקור DDS מסוג A/F/HDR/X, בונה אובייקטי PF, מעתיק את הנתונים, ומהדר ומריץ תוכנית header שנוצרה. עוזר בזמן בנייה עבור צינור ההמרה.
UNSNDWEB.cbl תת-תוכנית נקראת COBOL (ILE / OPM) מנוע מסכי WEB (הכותרת בעברית ניהול בניית מסכי WEB). פרמטר יחיד בגודל 64,412 בתים, מנתב גדול לפי קוד פעולה, בניית/פירוק XML, ותעבורת data-queue לשכבת ה-web. 170 סקשנים, כ-9,200 שורות.
⚠️ אזהרת שחרור משותף (מתוך כותרת המקור של UNSNDWEB, פרויקט "הגדלת שדות ל-K64"): יש לשחרר את UNSNDWEB יחד עם UNPARTQU, UNQUEST, UNSCRWEB, UNSNDA2C, UNSNDC2X ותוכניות התקשורת WBAPP, WBSRVR03, WBSRDQ. מכיוון שמבנה החוצץ המשותף השתנה, פריסה של UNSNDWEB לבדו תשבש את חילופי הפרמטרים.

🔗 קשרים — מי מריץ את מי

שלושת הקבצים האלה אינם שרשרת קריאה אחת. הם שייכים לשכבות שונות של אותה מערכת: תפריט אינטראקטיבי, כלי שירות בזמן בנייה, ומנוע בשכבת השרת.

📗 שכבת תפריט — M.rpg

M קורא את קבצי התפריט/ההרשאות/המשתמשים, ובבחירה תקינה ומורשית מבצע CALL לתוכנית הנבחרת (או פותח תת-תפריט). הוא אינו קורא ישירות ל- UNSNDWEB או ל-UNFILE2PCC; התוכניות שהוא מפעיל הן קטלוג האפליקציות הכללי.

📘 שכבת WEB — UNSNDWEB.cbl

UNSNDWEB נקרא על ידי שכבת שרת ה-web (WBSRVR03/WBAPP). הוא מחליף XML עם ה-front-end בדפדפן דרך data queue (באמצעות WBSRDQ), ובעצמו מבצע CALL לכ-40 תוכניות עזר UN* כדי לקרוא הגדרות מסך/פקודה ולבנות/לפרק את ה-XML.

📙 שכבת בנייה — UNFILE2PCC.cl

UNFILE2PCC הוא כלי שירות בזמן בנייה: הוא קורא ל-UNCRTHDRK כדי לייצר DDS ותוכנית לעדכון header, ואז מהדר ומריץ את UNUPDHDRK שנוצרה. הוא מופעל על ידי כלי ההמרה, ולא על ידי M או UNSNDWEB בזמן ריצה.

📗 M.rpg — מנהל התפריט

שם התוכנית ומשמעותו

תוכנית M — התפריט האינטראקטיבי של אפליקציית "ME" (תפריט). היא מציגה תפריט בתחנת עבודה 5250, מאמתת את האפשרות ואת ההרשאות של המשתמש, כותבת יומן שימוש, ומעבירה שליטה לתוכנית או לתת-התפריט הנבחרים. RPG III / RPG 400, 68 שגרות משנה.

📙 UNFILE2PCC.cl — כלי לבניית DDS / קבצים

שם התוכנית ומשמעותו

תוכנית UNFILE2PCC — "UN[iversal] FILE to PC[C]". בהינתן קובץ פיזי, היא מבצעת הנדסה לאחור ל-DDS של הקובץ ובונה את כל הפריטים שצינור ההמרה צריך: PF מקור זמני עם ארבעה חברים (A / F / HDR / X), ארבעה אובייקטי PF, עותק של הנתונים, ותוכנית header שנוצרה, מהודרה ומורצת.

פרמטרים ומשתנים

שםסוגתרגום לעבריתתיאור
&FROMFILE*CHAR 10קובץ מקור (קלט)הקובץ הפיזי שיש לעבד.
&FROMLIB*CHAR 10ספריית מקור (קלט)הספרייה המכילה את הקובץ.
&RET*CHAR 1דגל החזרה (פלט)נקבע ל-'1' אם לא ניתן ליצור את PF המקור או שהקובץ אינו קיים.
&TYPE*CHAR 6סוג אובייקטקבוע ל-'PF'.
&DDSFIL / &DDSLIB*CHAR 10קובץ / ספריית מקור DDSQDDSSRC ב-QTEMP.
&DDSMBR*CHAR 10שם חבר בסיס5 התווים הראשונים של &FROMFILE (%SST(&FROMFILE 1 5)).
&DDSMBRA/F/H/X*CHAR 10וריאנטים של חברבסיס + סיומת A / F / HDR / X.
&USER*CHAR 10משתמש נוכחינשלף בהתחלה.
&TEXT / &TEXTX / &MSG*CHAR 50/50/70עבודת טקסט / הודעהטקסט חבר ומחרוזות הודעות שגיאה.

זרימת קבצי עבודה (QTEMP)

שלבי הצינור (תוויות)

תוויתמה היא עושה
STARTשליפת המשתמש הנוכחי; אתחול משתני עבודה; קביעת &TYPE='PF', קובץ DDS בשם QDDSSRC ב-QTEMP, חבר = 5 התווים הראשונים של הקובץ.
CRT_SRCFCRTSRCPF QTEMP/QDDSSRC; בשגיאה קביעת &RET='1' ויציאה.
CHK_FILECHKOBJ על קובץ המקור; אם CPF9801 (לא נמצא) שליחת הודעה בעברית "קובץ אינו קיים", קביעת &RET='1', יציאה.
(חברים A/F/HDR/X)עבור כל וריאנט: CHKOBJ על החבר, ובהיעדרו ADDPFM אותו עם טקסט מתאר.
DSPFD / DSPFFDשפיכת תיאור הקובץ ותיאורי השדות אל QTEMP/DSPFD ו-QTEMP/DSPFFD.
OVRDBF1 / CRTDDSביטול (override) קבצי העבודה וחברי המקור, ואז CALL UNCRTHDRK לייצור מקור DDS/header. DLTOVR לאחר מכן.
CRT_DDSA/F/H/XDLTF+CRTPF לכל אחד מארבעת אובייקטי ה-PF מהחברים שנוצרו (SIZE(*NOMAX) LVLCHK(*NO)), ואז CLRPFM.
CPY_F2FCPYF את הנתונים לקובץ "F" (MBROPT(*REPLACE) FMTOPT(*MAP *DROP)).
DSPFD2 / OVRDBF2 / CMP_HDRPGM / UPDHDRתיאור קובץ ה-HDR שוב, ביטול, הידור UNUPDHDRK שנוצרה והרצתה.
EXITסיום התוכנית.
משתמשת ב-MONMSG CPF0000 בשפע כדי לבלוע מצבי "כבר קיים / לא נמצא", כך שהרצה חוזרת של הכלי היא אידמפוטנטית. השגיאות היחידות שהיא מציפה הן השתיים שחוסמות התקדמות: כשל ביצירת PF המקור וקובץ קלט חסר.

📘 UNSNDWEB.cbl — מנוע מסכי WEB

שם התוכנית ומשמעותו

תוכנית UNSNDWEB = "UN[iversal] SEND to WEB". תוכנית COBOL יחידה עם פרמטר כניסה אחד (חוצץ בגודל 64,412 בתים SND-DATA ש-חמישה copybooks של פרמטרים מגדירים אותו מחדש) ו- EVALUATE גדול על קוד פעולה WB-KOD-PEULA. היא בונה XML עבור מסכים/שדות/פקודות/גרידים/שאלונים/מכתבים, מחליפה אותו עם שכבת הדפדפן דרך data queues, ומפרקת את תשובת ה-XML חזרה לחוצץ המסך של האפליקציה.

DATA DIVISION — פרמטר הכניסה וה-overlays

התוכנית מקבלת פרמטר אחד בדיוק, SND-DATA, ומפרשת אותו מחדש באמצעות חמישה REDEFINES overlays — כל אחד נתמך על ידי copybook IN QWRKSRC. פריסות השדות המפורטות שוכנות אפוא ב-copybooks האלה; מה שהמקור עצמו מציג הוא החוצץ וחיווט ה-redefines.
