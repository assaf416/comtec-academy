🗂️ Files in this folder

These three sources belong to an AS/400 → UNIX portability toolkit. Each program carries dual code paths — lines tagged X&FLTA compile for AS/400, lines tagged X&FLTU compile for UNIX. They are peer utilities of the same toolkit, not a direct call chain among themselves.

FileTypeLanguagePurpose
UNRDRADD.cbl Service subprogram (called) COBOL (ILE, AS/400 & UNIX) Generic keyed-file READ routine for the indexed file UNRECADD. Called with an action code + key, returns one record + status. Builds the physical file name dynamically.
UNCHKWEB#P.rpg Service subprogram (called) RPG III / RPG 400 (fixed format) Environment & authorization check. Detects AS/400 vs WEB mode, loads the permission table from data areas, maintains an "entry-30" flag, returns mode + a permission code.
UNCVTD2C.cl Launcher / driver program CL (Control Language) "Convert file to CSV" launcher. Calls the external program UNFIL2CSV and sends the produced document's IFS address back as a program message.

Relationships — who runs who

Honest note: the three programs are independent utilities of the same toolkit. There is no direct call chain among them. UNCVTD2C calls the external program UNFIL2CSV (not present in this folder). UNRDRADD is a service routine that higher-level programs call. UNCHKWEB#P is called by apps/menus that need the environment + authorization state.

📘 UNRDRADD.cbl — keyed-file READ routine

Program name & meaning

UNRDRADD = UN (toolkit prefix) + RD (read) + RADD (record-address file UNRECADD) — a generic READ routine for the UNRECADD file.

Hebrew banner: רוטינת READ קובץ UNRECADD — בונה שם דינמי לקובץ — "READ routine for file UNRECADD — builds the file name dynamically."

Business role: a reusable access-method wrapper around the indexed file UNRECADD. Higher-level programs pass an action code (UN-PEULA) plus a three-part key (library + file + record name) and get back one record (LINK-REC) plus a status (UN-RETURN). The file is opened once and its physical name is built dynamically (<file>.RAD, optionally prefixed by an IFS path from env var TISTEMP_DDS on UNIX).

PROCEDURE DIVISION — every section

CBL 000-MONITOR — Main dispatcher / entry
Params: USING UN-PEULA, UN-RETURN, LINK-KEY, LINK-REC
Called by: program entry
Calls: ZA00-INIT002-CLOSE001-INIT100-MAIN

Description: The routine's driver. On UNIX first display it performs ZA00-INIT. If the action is close (UN-PEULA-CLOSE-FILE), it closes the open file and exits. Otherwise it performs 001-INIT (open once + save key) then 100-MAIN (dispatch), then EXIT PROGRAM.

IF UN-PEULA-CLOSE-FILE
   IF SW-FILE-OPEN-ON  PERFORM 002-CLOSE  END-IF
   GO TO 998
END-IF.
PERFORM 001-INIT.
PERFORM 100-MAIN.
998. EXIT PROGRAM.
CBL 001-INIT — Open once & save key
Called by: 000-MONITOR
Calls: 002-CLOSE (UNIX, if lib/file changed) 003-BUILD-NAME-DATABASE (UNIX)

Description: Opens the file the first time (building its dynamic name on UNIX), resets UN-RETURN to '00', and copies the caller's LINK-KEY parts into the record key. On UNIX it re-opens if the library/file changed since the last call.

IF SW-FILE-OPEN-OFF
   *> PERFORM 003-BUILD-NAME-DATABASE (UNIX)
   OPEN INPUT UNRECADD-FILE
   SET SW-FILE-OPEN-ON TO TRUE
END-IF.
MOVE '00' TO UN-RETURN.
MOVE LINK-KEY-LIB-NAME  TO UN-RADD-LIB-NAME.  *> +file +rec
CBL 002-CLOSE — Close the file
Called by: 000-MONITOR001-INIT (UNIX)
Calls: —

Description: Sets SW-FILE-OPEN-OFF and issues CLOSE UNRECADD-FILE.

SET SW-FILE-OPEN-OFF TO TRUE.
CLOSE UNRECADD-FILE.
CBL 003-BUILD-NAME-DATABASE — Dynamic file name (UNIX)
Called by: 001-INIT (UNIX path)
Calls: —

Description: Builds the physical file name as <file>.RAD into EZ-DATABASE. It reads env var TISTEMP_DDS into LIB-PATH; if blank it uses the plain name, otherwise it prefixes the IFS path (path/<file>.RAD) into DATABASE-UNRECADD.

       '.RAD' DELIMITED BY SIZE INTO EZ-DATABASE.
*> DISPLAY "TISTEMP_DDS" UPON ENVIRONMENT-NAME (UNIX)
IF LIB-PATH = SPACE   MOVE EZ-DATABASE TO DATABASE-UNRECADD
ELSE STRING LIB-PATH '/' EZ-DATABASE INTO DATABASE-UNRECADD END-IF.

Calls: —

Description: After EOF, re-reads the LAST RECORD (AS/400) to leave the cursor on the last row so a subsequent read-prior works.

Called by: 106-READ-PRIOR
Calls: —

Description: After begin-of-file, blanks the key parts, positions at the first record (key ≥ spaces) and reads next, so the cursor sits at the top.

Called by: 100-MAIN111 / 112 -*-MOVE

📗 UNCHKWEB#P.rpg — environment & authorization check

Program name & meaning

UNCHKWEB#P = UN (toolkit) + CHK (check) + WEB + #P (program / service suffix) — "check whether we run under WEB (vs AS/400) and load authorizations."

Business role: RPG III / RPG 400 mainline (C-specs, not COBOL divisions). Apps and menus call it to learn the run environment and to obtain the authorization (HARSHAOT) table. It reads three data areas — UNDDSWEB (environment flag), UNMAINDA (permission table), UNTDIDA (entry-30 flag) — populates the SYS array and returns the mode plus the first permission code.

PROCEDURE — single mainline broken into logical steps

RPG Step 1 — Close request — PARM01 = 'C'
Called by: any app / menu closing the service
Calls: —

Description: If the caller passes PARM01 = 'C', set on LR (last record — end the activation) and RETRN immediately. No environment or auth work is done.

C PARM01 IFEQ 'C'
C        SETON                LR
C        RETRN
C        ENDIF
RPG Step 2 — First-time environment detect — SWFR blank
Reads: UNDDSWEB (DTAARA → TMPARA)

Description: Only on the first call (SWFR = ' '), set SWFR = '1', read data area UNDDSWEB into TMPARA. Indicator 81 (from the IN) selects AS/400 (IFAS='1') vs WEB (IFAS='0').

C SWFR  IFEQ ' '
C       MOVE '1' SWFR
C *NAMVAR DEFN UNDDSWEB TMPARA
C       IN   TMPARA            81
C  81   MOVE '1' IFAS          * AS/400
C N81   MOVE '0' IFAS          * WEB
RPG Step 3 — Load auth table (WEB mode) — UNMAINDA → USYS → SYS
Reads: UNMAINDA (DTAARA → UNMAIN)

Description: When WEB (IFAS = '0'): read UNMAINDA into UNMAIN; if present, move USYS31 into the USYS array (else blank it). Then loop entries 1..31 copying each non-blank USYS,J into SYS,J, and put INTFRM into SYS,32.

C IFAS  IFEQ '0'
C *NAMVAR DEFN UNMAINDA UNMAIN
C       IN   UNMAIN            81
C N81   MOVEA USYS31 USYS
C       DO 31 J
C USYS,J IFNE ' '  MOVE USYS,J SYS,J  ENDIF
C       ENDDO
C       MOVE INTFRM SYS,32
C       ENDIF
RPG Step 4 — Entry-30 maintenance — UNTDIDA / UNMAIN
Reads/updates: UNTDIDA (DTAARA → UNTDID)UNMAINDA (UNMAIN)

Description: Blank USYS30 and SYS,30 (a 'T' is auto-seeded in multi-environment). Read UNTDIDA into UNTID; if the data area is absent (indicator N81), lock and read UNMAIN, set entry 30 = 'T' in both USYS30 and SYS,30, and write UNMAIN back.

C       MOVE ' ' USYS30
C       MOVE ' ' SYS,30
C *NAMVAR DEFN UNTDIDA UNTDID
C       IN   UNTDID            81
C N81 *LOCK IN UNMAIN
C N81   MOVE 'T' USYS30
C N81   MOVE 'T' SYS,30
C N81   OUT  UNMAIN
RPG Step 5 — Return parameters (every call) — PARM01 / PARM02
Called by: every entry
Calls: —

Description: On every call: move IFAS into PARM01; if more than one parameter was passed (MISPAR > 1), move SYS,1 into PARM02; then RETRN.

C       MOVE IFAS PARM01
C MISPAR IFGT 1
C       MOVEA SYS,1 PARM02
C       END
C       RETRN

📙 UNCVTD2C.cl — file-to-CSV launcher

Program name & meaning

UNCVTD2C = UN (toolkit) + CVT (convert) + D2C (data / document → CSV) — a thin CL launcher for "convert file to CSV."

Business role: a minimal CL driver. Given a file, library and document name it calls the external program UNFIL2CSV, receives the produced document's IFS address, and sends it back as a program message. It does not do the conversion itself.

PROCEDURE — labelled steps

CL START — declarations & PGM interface
Params: PGM(&FILE &LIB &DOCNAM)
Called by: operator / calling program
Calls: —

Description: Declares the three received parameters plus the work variables &ADDR (120) and &RET (1).

PGM (&FILE &LIB &DOCNAM)
   DCL &RET *CHAR 1
   DCL &FILE *CHAR 10
   DCL &LIB  *CHAR 10
   DCL &DOCNAM *CHAR 30
   DCL &ADDR *CHAR 120
CL CALL — clear work vars & run UNFIL2CSV
Called by: START (fall-through)
Calls: UNFIL2CSV (external)

Description: Clears &ADDR and &RET, then calls the external converter UNFIL2CSV passing all five parameters. UNFIL2CSV returns the document's IFS path in &ADDR and a status in &RET.

CALL:
   CHGVAR &ADDR ' '
   CHGVAR &RET  ' '
   CALL PGM(UNFIL2CSV) PARM(&FILE &LIB &DOCNAM &ADDR &RET)
CL SNDPGMMSG — report the document address; EXIT
Called by: CALL (fall-through)
Calls: —

Description: Sends the document's IFS address back to the caller as a program message, then ENDPGM at the EXIT label.

   SNDPGMMSG MSG('Document''s address : ' || &ADDR)
EXIT:
   ENDPGM
🔎 &RET is captured but not tested in this launcher — the value the program surfaces to the caller is &ADDR (the document's IFS path).
