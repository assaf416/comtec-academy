Relationships — who runs who

A business application calls SAATCGSL passing the transaction record plus a small options block (language 1=Hebrew/2=English and configuration 1=collection/2=claim). SAATCGSL fetches a running number via SAGETBKR, trims each input field with DCSTRING, packs the values into length-prefixed blocks, and calls the RPGLE engine SAATCG. SAATCG reads its credentials/URL from SAMAINPF, builds the doDeal SOAP request from the same 703-byte record, POSTs it to the CreditGuard server, parses the XML reply, writes the result back into the record and into its output block, and logs the whole exchange. Control returns up the chain and SAATCGSL unpacks the response for the caller. The 703-byte record is the backbone: COBOL names its fields DCCGE-xxx, RPGLE names the same bytes PTxxx.

Other programs called along the way: SAGETBKR (running number), DCSTRING (string trim/compress), SAATCGSL#P (session/SSID change), TBERR (error-message text), and — inside SAATCG — CONVERT (EBCDIC-424 ↔ UTF-8-1208 code-page conversion) and the HTTPAPI service program.

CBL saatcgsl.cbl — program SAATCGSL

Program name & meaning

SAATCGSL = SA (the SA-series application prefix) + ATCG ("Ashrait / CreditGuard") + SL (the "SeLuka" / wrapper-service suffix). It is a thin, defensive wrapper in front of the RPGLE API engine SAATCG. Its business job is to let any in-house IBM i program run a credit-card transaction without knowing anything about SOAP, HTTP, XML, varying-length RPGLE parameters or code-page conversion. The header comment (Hebrew) states its five responsibilities:

  • Monitor / wrap the call to the CreditGuard API.
  • Validate parameter correctness.
  • Build the parameters for the API.
  • Call the API.
  • Process the parameters that come back from the API (plus add USER DATA, e.g. policy branch).

ERD 1 — the wrapper's parameter model

The wrapper never talks to CreditGuard directly. It maps the caller-friendly SAATCG-1/2-PARAM into the RPGLE-friendly length-prefixed SAATCGR-1/2 and DCCGTRNE-IO blocks, calls the engine, then copies the results back.

RPG saatcg.rpgle — program SAATCG
